I built this hands-on automation framework from scratch using Playwright, TypeScript, and GitHub Actions to mimic a real-world, production-ready QA setup. Instead of sticking to basic tutorial code, I implemented custom fixtures in fixtures/index.ts to automatically handle page object instantiation and teardown, keeping the test specs clean and boilerplate-free. I also put business-logic assertions directly inside the Page Object Models (like loginPage.expectSuccessfulLogin()) so that if a selector or redirect route changes, I only have to update it in a single file. To optimize developer experience versus CI stability, I configured the framework to fail instantly with zero retries during local development, but allowed up to two retries in GitHub Actions to account for cloud network blips (plus a forbidOnly check to block accidental test.only() commits from skipping the pipeline). The suite covers data-driven UI authentication flows (using parameterized loops for negative test cases), form interactions, and backend API CRUD validation using Playwright's headless request context, all running fully in parallel across a Chromium and Firefox build matrix. To run it locally, just clone the repo, run npm install && npx playwright install --with-deps, and execute npx playwright test (or add --ui to open the interactive debugger).
